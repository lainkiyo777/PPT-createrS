# 延长风电数据与预测模型汇报 PPT 内容大纲

## 汇报主线

本套 PPT 建议围绕一条完整闭环展开：

```text
5 分钟 SCADA 原始数据
  -> 数据质量与字段体系
  -> 阶段一：未来 15 分钟至 4 小时风速、风向联合预测
  -> 阶段二：给定风况到功率的映射预测
  -> 接入闭环：未来风况预测 + 功率映射 = 未来风功率预测
```

建议总页数控制在 **22 页左右**。前三部分分别对应用户已有三份材料：数据集介绍、风速风向预测、功率预测；最后增加一组“闭环衔接与下一步”页面，把三部分串成完整故事。

## 第二部分：风速风向预测

### 第 8 页：阶段一任务定位

**核心结论：** 阶段一目标是建立未来 15 分钟至 4 小时的风速、风向联合预测模块，为后续功率预测提供风况输入。

**任务定义：**

```text
输入：过去 24 小时多变量 SCADA 序列 + 风机 ID/类型 + 坐标/地形位置编码
输出：未来 16 个 horizon 的风速和风向
时间范围：15min、30min、...、240min
```

**讲解口径：** 阶段一不直接预测功率，而是先预测未来风况；这是后续功率闭环预测的第一步。

**版式建议：** 流程图：历史 SCADA -> 时序预测模型 -> 未来风速/风向。

### 第 10 页：预测任务与评价指标

**核心结论：** 阶段一是多步、多目标预测，同时输出风速与风向。

**任务设置：**

| 项目 | 设置 |
|---|---|
| 时间粒度 | 5min |
| 历史输入 | 288 steps，即 24h |
| 预测步长 | h3-h48 |
| 实际预测时间 | 15min 至 240min |
| 风速目标 | wind_speed_mean_clean |
| 风向目标 | wind_dir_sin、wind_dir_cos |
| 输出维度 | 16 horizons x 3 targets |

**评价指标：**

- 风速：MAE / RMSE，单位 m/s
- 风向：Direction cosine、周期角度 MAE，单位 deg
- 风向采用 sin/cos 表示，避免 0° 和 360° 边界问题

**版式建议：** 左侧任务定义，右侧指标解释；风向可放圆环示意。

**讲解口径：** 先建立简单 baseline，再用复现模型对齐外部方法，最后展示我们的方法在统一测试集下的表现。

## 第三部分：功率预测

### 第 16 页：阶段二任务定位与数据

**核心结论：** 阶段二解决“给定目标时刻风况，如何映射为功率”的问题。

**任务链路：**

```text
第一阶段：未来风速、未来风向预测
  -> 第二阶段：风速、风向、温度、风机编号
  -> 输出：目标时刻有功功率
```

**数据数字卡片：**

- 62,537：清洗后 5min 样本
- 4 台：参与建模风机
- 6250kW：单机额定功率
- 43,775 / 9,381 / 9,381：训练 / 验证 / 测试样本
- 8,874：正常工况测试样本

**讲解口径：** 阶段二不是重新做时序预测，而是学习“风况 -> 功率”的映射关系。

### 第 17 页：阶段二方法路线

**核心结论：** 从传统机器学习到深度表格模型，再到多工况专家模型，最终 MoE 表现最优。

**路线：**

```text
传统机器学习：LightGBM / XGBoost / RandomForest
  -> 深度表格模型：MLP / DeepFM / Residual MLP / Wide & Deep / FT-Transformer-lite
  -> 特征与训练改进：风机 Embedding / 高功率加权 / 正常工况筛选 / 衍生特征
  -> 结构改进：MoE 多工况专家 / 灰盒功率曲线+残差 / 每风机独立 head
```

**有效改进：**

- 风机 Embedding：刻画不同机组个体差异
- 风向 sin/cos：处理角度周期连续性
- 风速²、风速³：增强功率曲线非线性表达
- 高功率加权：提升中高功率段拟合能力
- MoE 多工况模型：适合不同风况下不同功率映射规律

### 第 18 页：功率映射模型结果对比

**核心结论：** MoE 在全测试集和正常工况下均为当前最优模型。

**结果表：**

| 模型 | 全测试集 MAE(kW) | 正常工况 MAE(kW) | 全测试集 R² | 正常工况 R² | 全测试集 NMAE | 全测试集 WAPE | 15min 合格率 |
|---|---:|---:|---:|---:|---:|---:|---:|
| XGBoost 高功率加权 | 208.83 | 181.80 | 0.9186 | 0.9462 | 3.34% | 17.65% | - |
| MLP + Embedding + 加权 | 182.81 | 158.21 | 0.9296 | 0.9547 | 2.92% | 15.45% | 84.32% / 87.47% |
| FT-Transformer-lite + 加权 | 183.44 | 157.47 | 0.9301 | 0.9560 | 2.94% | 15.50% | - |
| 灰盒功率曲线 + 残差 | 183.63 | 155.54 | 0.9261 | 0.9536 | 2.94% | 15.52% | - |
| **MoE 多工况模型** | **179.17** | **154.31** | **0.9323** | **0.9573** | **2.87%** | **15.14%** | **84.83% / 88.04%** |

**讲解口径：** XGBoost 稳定但误差偏高；MLP、FT-Transformer 和灰盒残差模型接近；MoE 在主要指标上当前最优。

### 第 19 页：最优模型 MoE 结构

**核心结论：** MoE 通过“多专家 + 门控选择 + 风机个体表示”适配不同风况下的功率曲线差异。

**结构说明：**

```text
风速、风向、温度、衍生特征 + 风机 Embedding
  -> 特征向量 z
  -> Gate 输出不同 Expert 权重
  -> 多个 Expert 分别预测
  -> 加权融合得到功率预测
```

**关键公式：**

```text
g(z) = Softmax(Gate(z))
P_hat = 6250 x sum(g_k(z) * Expert_k(z))
```

**数字卡片：**

- 179.17kW：全测试集 MAE
- 154.31kW：正常工况 MAE
- 0.9323：全测试集 R²
- 0.9573：正常工况 R²
- 88.04%：正常工况 15min 合格率

**版式建议：** 中间放 MoE 结构图，右侧放结果数字卡片。

### 第 20 页：误差分布与高风速难点

**核心结论：** 当前误差主要集中在高功率 / 高风速区，后续优化应重点处理这一段。

**现象：**

- 低风速区预测较稳定，MAE 较低
- 中等风速区整体趋势拟合较好
- 8m/s 以上高风速区误差明显增大
- 9-12m/s 区间误差最高，可能达到 600-1000kW 量级

**可能原因：**

- 高功率样本数量相对较少
- 接近额定功率后控制策略影响增强
- 桨距控制、限电、偏航误差、设备状态会影响功率
- 相同风速下，不同风机和不同工况的功率差异更大

**后续优化：**

- 加强高功率样本权重
- 对高风速 / 高功率区单独建模或校准
- 引入更细粒度专家划分
- 对额定功率附近加入物理约束

**版式建议：** 左侧放误差随风速区间变化图，右侧放“现象、原因、优化”三张小卡片。

### 第 21 页：阶段二结论与闭环衔接

**核心结论：** MoE 已经解决给定风况到功率映射的核心问题，下一步要接入阶段一预测风况形成闭环。

**当前结论：**

- 阶段二当前最优模型：MoE 多工况专家模型
- 全测试集 MAE = 179.17kW
- 正常工况 MAE = 154.31kW
- 正常工况 R² = 0.9573
- 正常工况 15min 合格率 = 88.04%

**与阶段一衔接：**

```text
阶段一输出：未来风速、未来风向
  -> 输入 MoE：未来风速、未来风向、温度、风机编号
  -> MoE 输出：未来目标时刻功率
  -> 形成未来 15min-4h 风功率预测链路
```

**讲解口径：** 阶段一解决“未来风况是什么”，阶段二解决“这个风况对应多少功率”，两者合起来才是完整的未来功率预测。

## 第四部分：闭环衔接与下一步

### 第 22 页：完整闭环方案

**核心结论：** 后续落地应从单模块验证走向端到端闭环验证。

**闭环流程：**

```text
原始 SCADA 数据
  -> 数据清洗与特征构造
  -> 阶段一 PatchTST / FuXi-CFD 预测未来风速风向
  -> 阶段二 MoE 映射未来功率
  -> 输出未来 15min、30min、...、240min 功率
  -> 按误差、合格率、考核口径评估
```

- 限电记录

### 第 23 页：下一步验证计划

**核心结论：** 下一阶段重点是验证模型鲁棒性、打通闭环，并按甲方考核口径评估。

**短期计划：**

- 对 full test 最优 PatchTST 和 FuXi-CFD best 补充多随机种子验证
- 将 OpenOA clean 微调 checkpoint 回评 full test，确认完整工况鲁棒性
- 输出分 horizon、分风机、分风速区间、分风向扇区误差诊断图
- 接入阶段一预测结果到 MoE，重新计算闭环后的 MAE、RMSE、R²

**中期计划：**

- 优化 PatchTST 的 patch_len、stride、d_model 等参数
- 对 FuXi-CFD 坐标和地形特征做严格消融
- 将 clean 样本筛选从单一策略扩展为 strict / moderate / soft 三档
- 强化高功率区预测，包括高功率加权、分段专家和物理约束

**考核口径：**

- 计算 ±25% 偏差阈值下的合格率
- 计算低出力 3% 免考核后的触发比例
- 估算偏差积分电量和考核费用

### 第 24 页：总括结论

**核心结论：** 当前工作已经完成从数据资产、风况预测到功率映射的主要技术链路，具备进入闭环验证和现场口径对齐的基础。

**三点总结：**

1. **数据基础可用**：5 分钟 SCADA 数据字段完整、无空值、无重复，可支撑预测建模；但当前样本实际覆盖 4 台机组，全场口径需进一步确认。
2. **风况预测链路已建立**：阶段一完成 full test 与 OpenOA clean 两类口径实验，PatchTST 当前 full test Speed MAE = 1.3299，clean 微调后 Speed MAE = 1.2486。
3. **功率映射模型已形成**：阶段二 MoE 为当前最优，正常工况 MAE = 154.31kW，R² = 0.9573，15min 合格率 = 88.04%。

**收尾话术：**

> 本次工作已经把底层数据、风况预测和功率映射三部分打通。下一步重点不是单纯继续堆模型，而是把阶段一预测风况真正接入阶段二 MoE 功率映射，按完整工况和甲方考核口径验证未来风功率预测效果，并结合状态、告警、限电和检修数据完成误差归因。
