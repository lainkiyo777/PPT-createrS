## 2. 为什么没有询问用户

本次没有询问用户，不是因为环境不支持 GUI，而是因为配置在生成前已经被程序设置为 `workflow_mode: auto` + `selection_mode: direct`。生成报告还把“用户提供蓝色模板”解释为“沿用模板唯一视觉路线”。这把视觉素材来源误当成了配置选择结果。

`deck-brief.yaml` 中的品牌约束“仅使用用户提供的蓝色模板”只能说明素材范围，不能证明用户已经选择了：

- 演示类型；
- 视觉风格；
- 演示效果；
- 输出模式；
- 备注模式；
- 时长和信息密度。

## 3. 是否错误地把上传模板视为用户已完成选择

**是。** 证据有三处：

1. `deck-config.yaml` 的 `classification_reason` 把模板存在直接转换成“沿用模板唯一视觉路线”。
2. `selected-style.yaml` 只有 `source_template`、`route`、palette、fonts 和复用 rules，没有 `selected_by: user` 或确认时间。
3. `generation-report.md` 记录了“template-following”并直接进入 24 页渲染，没有 configuration awaiting 状态或用户确认事件。

模板可以是候选视觉来源，也可以是用户明确指定的品牌约束，但不能替代 P0-A 配置确认，更不能替代 Manual 候选选择。

## 4. 是否生成了 selected-style.yaml

**生成了。** 文件是：

```text
projects/wind-blue-template-v1.1/output/selected-style.yaml
```

它记录了模板来源、路线、颜色、字体和“只替换继承文本框”的规则，但没有记录：

- `selected_candidate`
- `selected_by`
- `confirmation_timestamp`
- `candidate_profile_path`

因此它不满足新的 Manual Gate。

## 5. selected-style.yaml 是谁确认产生的

**不是用户确认产生的，是程序自行生成的。** 当前文件没有用户确认元数据；output 目录也没有 `deck-config.confirmed.yaml`、`build-status.json` 或候选选择日志可以证明用户确认过。`style-candidates/` 也没有形成 2–4 套可选择候选包。

## 6. 是否存在真正暂停等待用户输入的代码路径

**在本次 v1.1 执行时不存在。** 审计到的仓库原有路径只有：

- `SKILL.md` 对 manual/guided 的文字说明；
- `validate_p0.py` 对 `selected-style.yaml` 和候选目录的静态检查；
- 生成脚本对模板 starter 的导入、编辑、渲染和合成。

没有运行时 `input()`、编号选项、`awaiting_configuration`、`awaiting_style_selection` 或“无确认则退出”的生产 Gate。`generation-report.md` 是唯一可识别的执行记录，仓库中没有真实的交互日志。因此流程可以直接从 config 进入 outline、slide-spec、preview 和 PPTX。

## 7. 修复后的行为

本次新增 `skills/ppt-creaters/scripts/config_gate.py`，并接入 P0 contract 文档和 validator：

- guided 未确认：写入 `build-status.json` 和 `config-selection-report.md`，状态为 `awaiting_configuration`，不写 `deck-config.confirmed.yaml`。
- manual 未选候选：状态为 `awaiting_style_selection`，不写 `selected-style.yaml`。
- `selected_by` 不是 `user`：直接失败。
- 上传模板只作为 `template_source`，不能跳过配置或候选确认。
- auto 可以不询问，但必须写 `confirmation_method: auto_inference` 并在报告中记录推断。
- direct 只有在调用者证明所有必要字段都由用户显式提供时才可确认。
- 所有 outline、slide-spec、图片、PPTX 和 publish 阶段必须先调用 `assert_generation_allowed(output, stage)`。
